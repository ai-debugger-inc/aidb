# AI Debugger Release Notes: 0.0.9

**GitHub Release**:
[0.0.9](https://github.com/ai-debugger-inc/aidb/releases/tag/0.0.9)

## Overview

Java remote attach improvements with automatic source detection, MCP response limiting for high-thread JVMs, and contributor documentation.

## Breaking Changes

- **Deprecated adapter tools removed**: `download_adapter`, `download_all_adapters`, and `list_installed_adapters` tools have been removed
  - Use the consolidated `adapter` tool with action parameter instead:
    - `adapter(action='download', language='python')` replaces `download_adapter`
    - `adapter(action='download_all')` replaces `download_all_adapters`
    - `adapter(action='list')` replaces `list_installed_adapters`

## New Features

- **Java Source Auto-Detection**: Automatic source path discovery for Maven/Gradle projects when remote debugging JVMs
  - Scans workspace for `pom.xml` and `build.gradle` files
  - Resolves JAR-internal paths (e.g., `trino-main.jar!/io/trino/Foo.java`) to local sources
  - Enabled automatically when `workspace_root` is provided
- **Thread Limiting**: New `AIDB_MCP_MAX_THREADS` config option (default: 10) to limit thread responses for high-thread JVMs
  - Current thread is always prioritized in limited results
  - Truncation metadata added when limits apply
- **Child Variable Inspection**: New `get_child_variables()` method for inspecting nested variable structures

## Enhancements

- **Java Remote Attach**: Improved initialization sequence for remote attach mode
  - Lazy LSP-DAP bridge initialization (bypasses PRE_LAUNCH hook)
  - Proper java-debug plugin initialization before `startDebugSession`
  - Skip adapter process alive check for remote sessions
- **Consolidated Adapter Tool**: Single `adapter` tool with action-based dispatch replaces three separate tools

## Bug Fixes

- Fixed missing `await` on `attach_remote()` in session lifecycle
- Fixed `dataId` field name (was incorrectly `data_id`) in breakpoint handlers
- Fixed `get_child_variables` return type to wrap in `AidbVariablesResponse`
- Fixed thread ID resolution for remote attach where IDs may not start at 1
- Replaced `StartRequestType` magic string with enum in state management

## Documentation

- Added contributor documentation (CONTRIBUTING.md, CODE_OF_CONDUCT.md, SECURITY.md)
- Added GitHub issue templates (bug report, feature request)
- Added GitHub pull request template
- Added full contributing guide at `docs/community/contributing.md`
- Updated Java debugging documentation with source auto-detection examples

## Internal Changes

- Removed deprecated `is_mcp_compact()` method; replaced with `is_mcp_verbose()`
- Added unit tests for source detection and response limiter
