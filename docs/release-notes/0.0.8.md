# AI Debugger Release Notes: 0.0.8

**GitHub Release**:
[0.0.8](https://github.com/ai-debugger-inc/aidb/releases/tag/0.0.8)

## Overview

Architecture improvements with new adapter components, parallel test stability fixes, and CLI simplification.

## New Features

- **TargetResolver**: Automatic target type detection (file/module/class) per language
- **SourcePathResolver**: Source path mapping for remote debugging scenarios
- **AdapterCapabilities**: Static capability discovery without spawning adapter processes
- **Output Collection**: Retrieve logpoint messages and program output via `get_output()`
- **Data Breakpoints**: Java watchpoint support for monitoring variable changes
- **Version Consistency**: New `versions check-consistency` CLI command

## Enhancements

- Renamed MCP server from `aidb-debug` to `ai-debugger` for brand consistency
- Consolidated timeout/port constants into centralized configuration
- Standardized logger instantiation via `aidb_logging` module
- CLI architecture simplified (5 services consolidated into CommandExecutor)
- Parallel test execution stability improvements

## Breaking Changes

- **MCP server name**: Changed from `aidb-debug` to `ai-debugger`
  - Users must update MCP client configuration
  - Tool prefixes change from `mcp__aidb-debug__*` to `mcp__ai-debugger__*`
- **CLI flags**: Test command flags aligned with pytest conventions
  - `--pattern` → `-k` (pytest -k style)
  - `--parallel` → `-n` (pytest -n style)
- **Removed**: `--profile` option and `test report` subcommand
